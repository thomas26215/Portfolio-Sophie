S4 : Refactoring d'une application communicante pour le suivi de la recherche de stage

L’objectif de ce projet était de refactoriser une application qui permet aux étudiants d'informer le responsable des stages de l'IUT de leur avancée dans leurs recherches. Pour cela, il nous a été donné le code source d'une application Android en JAVA pour les étudiants et une en Synfony pour le professeur référent des stages.

Travail de Groupe

Tout d'abord, nous devions identifier les différents problèmes et nous les avons hiérarchisés, par exemple, nous avons priorisé les bugs qui empêchaient d'utiliser l'application correctement. Ensuite, nous les avons attribués à chaque membre de l'équipe pour les corriger. Nous avons également proposé un nouveau schéma relationnel de base de données en forme 3 FN.

Travail Personnel

Je me suis occupée de la partie mobile avec un autre membre de l'équipe. J'étais également cheffe de projet, ce qui implique que je me suis occupée de l'organisation générale du groupe.

Ce projet m’a permis d’apprendre à comprendre le code d'une autre personne et de pouvoir le corriger et donc j'ai pu me rendre compte de l'impact qu'a la qualité de code sur la maintenabilité d'une application.

S2 : Développement d'une application d'aide à l'organisation d'événement en JAVA

L’objectif de ce projet était de réaliser une application lourde pour aider un organisateur d'événement en JAVA, JavaFX et FXML en équipe de 6. Nous devions choisir le type d'événement puis faire la conception puis réaliser le code.

Travail de Groupe

Nous avons tout d'abord choisi notre type d'événement : les vides-greniers. Nous avons trouvé un nom, un logo, les couleurs. Par la suite, nous avons en parrallèle rédigé le dossier de gestion de projet (avec en particulier les contraintes, les risques induits et la mitigation) et le dossier de conception UML (avec les diagrammes des cas d'utilisation, leurs priorisation).

Travail Personnel

Je me suis occupée de la partie communication, de la distribution des tâches dans la première partie de conception. J'ai fait quelques fonctions comme par exemple, le fait de bloquer et d'entourer en rouge les champs obligatoires dans le formulaire.

Ce projet m’a permis d’apprendre à faire du code dans une équipe, comprendre le code des autres et distribuer les tâches.

S1 : Classement automatique de dépêches

L’objectif de ce projet était de trier automatiquement des dépêches de presse dans différentes catégories comme, par exemple, sport, culture, politique… avec le plus haut pourcentage de réussite possible en optimisant au mieux notre programme.

Travail de Groupe

Nous avons réalisé ce projet en binôme en deux phases. Dans un premier temps en s’aidant de lexiques créés manuellement par catégorie, en associant des mots clés à un poids (entre 1 et 3). Les dépêches sont ensuite associées à une catégorie en calculant leurs poids pour chaque catégorie. Dans un second temps, les lexiques sont créés automatiquement en calculant la récurrence de mots d’autres dépêches qui ont une catégorie donnée. Pour ces deux phases, il fallait calculer le taux de réussite et optimiser les lexiques créés manuellement et optimiser le programme pour que ce taux soit le plus haut possible et faire un rapport détaillé en anglais pour expliquer notre code.

Travail Personnel

Je me suis occupée de réaliser tout le code et de l’optimiser au mieux et de le détailler à l’aide de nombreux commentaires.

Ce projet m’a permis de travailler avec une personne d'un niveau plus faible que le mien et donc d’expliquer mon code pour que mon camarade comprenne et puisse ensuite l’expliquer à son tour.
